[1mNAME[0m
       awk - pattern-directed scanning and processing language

[1mSYNOPSIS[0m
       [1mawk[0m [ [1m-F[0m [4mfs[24m ] [ [1m-v[0m [4mvar=value[24m ] [ [4m'prog'[24m | [1m-f[0m [4mprogfile[24m ] [ [4mfile[24m [4m...[24m  ]

[1mDESCRIPTION[0m
       [4mAwk[24m scans each input [4mfile[24m for lines that match any of a set of patterns specified literally in [4mprog[24m or in one or more files specified as [1m-f[0m [4mprogfile[24m.  With each pattern there can be an
       associated action that will be performed when a line of a [4mfile[24m matches the pattern.  Each line is matched against the pattern portion of every pattern-action statement; the associated action is
       performed for each matched pattern.  The file name [1m-[0m means the standard input.  Any [4mfile[24m of the form [4mvar=value[24m is treated as an assignment, not a filename, and is executed at the time it would
       have been opened if it were a filename.  The option [1m-v[0m followed by [4mvar=value[24m is an assignment to be done before [4mprog[24m is executed; any number of [1m-v[0m options may be present.  The [1m-F[0m [4mfs[24m option
       defines the input field separator to be the regular expression [4mfs[24m.

       An input line is normally made up of fields separated by white space, or by the regular expression [1mFS[0m.  The fields are denoted [1m$1[0m, [1m$2[0m, ..., while [1m$0[0m refers to the entire line.  If [1mFS[0m is null,
       the input line is split into one field per character.

       A pattern-action statement has the form:

              [4mpattern[24m [1m{[0m [4maction[24m [1m}[0m

       A missing [1m{[0m [4maction[24m [1m}[0m means print the line; a missing pattern always matches.  Pattern-action statements are separated by newlines or semicolons.

       An action is a sequence of statements.  A statement can be one of the following:

              if( [4mexpression[24m ) [4mstatement[24m [ else [4mstatement[24m ]
              while( [4mexpression[24m ) [4mstatement[24m
              for( [4mexpression[24m ; [4mexpression[24m ; [4mexpression[24m ) [4mstatement[24m
              for( [4mvar[24m in [4marray[24m ) [4mstatement[24m
              do [4mstatement[24m while( [4mexpression[24m )
              break
              continue
              { [ [4mstatement[24m [4m...[24m ] }
              [4mexpression[24m              # commonly [4mvar[24m [4m=[24m [4mexpression[24m
              print [ [4mexpression-list[24m ] [ > [4mexpression[24m ]
              printf [4mformat[24m [ , [4mexpression-list[24m ] [ > [4mexpression[24m ]
              return [ [4mexpression[24m ]
              next                    # skip remaining patterns on this input line
              nextfile                # skip rest of this file, open next, start at top
              delete [4marray[24m[ [4mexpression[24m ]# delete an array element
              delete [4marray[24m            # delete all elements of array
              exit [ [4mexpression[24m ]     # exit immediately; status is [4mexpression[24m

       Statements are terminated by semicolons, newlines or right braces.  An empty [4mexpression-list[24m stands for [1m$0[0m.  String constants are quoted " ", with the usual C escapes recognized within.
       Expressions take on string or numeric values as appropriate, and are built using the operators [1m+[0m [1m-[0m [1m*[0m [1m/[0m [1m%[0m [1m^[0m (exponentiation), and concatenation (indicated by white space).  The operators [1m![0m [1m++[0m [1m--[0m
       [1m+=[0m [1m-=[0m [1m*=[0m [1m/=[0m [1m%=[0m [1m^=[0m [1m>[0m [1m>=[0m [1m<[0m [1m<=[0m [1m==[0m [1m!=[0m [1m?:[0m are also available in expressions.  Variables may be scalars, array elements (denoted [4mx[24m[1m[[0m[4mi[24m[1m][0m) or fields.  Variables are initialized to the null string.  Array
       subscripts may be any string, not necessarily numeric; this allows for a form of associative memory.  Multiple subscripts such as [1m[i,j,k][0m are permitted; the constituents are concatenated,
       separated by the value of [1mSUBSEP[0m.

       The [1mprint[0m statement prints its arguments on the standard output (or on a file if [1m>[0m [4mfile[24m  or [1m>>[0m [4mfile[24m  is present or on a pipe if [1m|[0m [4mcmd[24m  is present), separated by the current output field
       separator, and terminated by the output record separator.  [4mfile[24m and [4mcmd[24m may be literal names or parenthesized expressions; identical string values in different statements denote the same open
       file.  The [1mprintf[0m statement formats its expression list according to the [4mformat[24m (see [4mprintf[24m(3)).  The built-in function [1mclose([0m[4mexpr[24m[1m)[0m closes the file or pipe [4mexpr[24m.  The built-in function
       [1mfflush([0m[4mexpr[24m[1m)[0m flushes any buffered output for the file or pipe [4mexpr[24m.
